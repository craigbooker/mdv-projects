Accessibility

There are many options for developers when it comes to making their web sites or applications accessible to everyone. For elements that would require text alternatives Recommendations for My App

  • Text Alternatives
  • Alt tags At the bare minimum all elements should include alt tags that provide value. Another route would be to include the longdesc attribute where possible. Make these tags meaningful.
  • Page Titles Using proper page titles for each part of the web site or application. Avoiding the use of pictures replacement via CSS or pictures of words. Replace any onmouse or onkey triggers with the onclick trigger.
  • Video Native HTML5 video controls that will allow users to take advantage of keyboard controls. Add descriptions, subtitles, and captions to videos through the use of vtt.
  • Forms For my crud app, I could make sure to use label elements to associate text labels with form controls.
  • Content Presentation
  • Arrange content in a linear order that makes sense when you can't see the entire page or layout. If you were to go to a
  • web site with only the voice over turned on would the structure make sense?
  • The use of scripting languages.
  • Browsers built for those with Accessibility needs often turn off scripting languages and do not include plug-ins. By trying to reduce the amount of plugins used within an application and by providing alternatives for browsers that don't support scripting. At this point in my application I could check to see if the browser supports scripting and if it doesn't, at least provide a notification of some sort that will be accessible by the user. There are many frustrations when it comes to users with accessibility needs and web services. Some of the common frustrations that users with accessibility needs experience are with applications such as flash. Without the proper approach by the content creator, flash content is often useless to those with visual impairment. Many times the video will be set to loop which causes the screenreader to get caught in a loop. Which, if the user does not know how to exit, can prove extremely frustrating.
  • iPhone/iPad
  • Apple goes above and beyond when it comes to accessibility on their devices. The voice over feature proves quite powerful for those with visual impairment. The rotor is a virtual control which allows users to navigate their device in different ways. These are set according to their preferences. The rotor allows users to navigate a page word by word or character by character. The rotor can also be used to navigate web pages. Another feature is Siri. Siri allows users to text, email, schedule meetings, set a reminder, and place phone calls via voice command. For those with partial visual impairment iOS devices allow users to zoom in on any part of the screen. It is also possible to switch the screen from black on white to white on black for higher screen contrast. While working for Apple, I kind of stumbled into becoming knowledgeable about accessibility options for Apple computers, and iOS devices. I would have customers who would come in looking for solutions that met their accessibility needs. I had customers who would tell others with accessibility needs to come in and ask for me. We would go over the features above to see how Apple products could enrich their lives.
